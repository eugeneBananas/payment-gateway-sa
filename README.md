# Payment Gateway Aggregator — System Analysis

Проектирование клиент-серверной системы платёжного шлюза-агрегатора для обработки онлайн-платежей.

## О проекте

Спроектировал концептуальную и логическую модели системы, которая объединяет несколько банков и платёжных провайдеров через единую точку интеграции. Система обеспечивает приём, маршрутизацию и антифрод-проверку онлайн-платежей.

## Что сделано

### Концептуальная модель
- Анализ предметной области и выявление участников системы (плательщик, мерчант, администратор)
- Сравнительный анализ существующих решений (CloudPayments, ЮKassa, PayU)
- Моделирование текущего бизнес-процесса **AS-IS** в нотации BPMN
- Проектирование целевого бизнес-процесса **TO-BE**
- Формулировка функциональных и нефункциональных требований
- Построение **Utility Tree** для нефункциональных характеристик
- Разработка **Use Case диаграммы**

### Логическая модель
- **Диаграмма классов** — ключевые сущности и их взаимосвязи
- **Диаграмма последовательности** — сценарий обработки платежа с антифрод-проверкой
- **Диаграмма состояний** — жизненный цикл платёжного запроса и соединения с банком
- **ER-диаграмма** — логическая модель данных (users, payments, transactions, antifraud_checks, notifications)
- **Архитектурная диаграмма в нотации ArchiMate** — микросервисная архитектура с API Gateway

### Архитектурные диаграммы C4
- System Context Diagram
- System Landscape Diagram
- Container Diagram

## Стек и инструменты

`BPMN` `UML` `C4 Model` `ArchiMate` `StormBPMN` `draw.io` `PostgreSQL` `REST API`

## Нефункциональные требования системы

| Характеристика | Значение |
|----------------|----------|
| Время отклика (p95) | ≤ 150 мс для авторизации |
| Доступность | ≥ 99.9% в месяц |
| Пиковая нагрузка | ≥ 500 RPS |
| Безопасность | TLS 1.3, AES-256, JWT, PCI DSS |
